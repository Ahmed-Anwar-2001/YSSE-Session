01. WHAT IS IT?

THE RUSSIAN DOLL ANALOGY

THE OUTER SHELL ARTIFICIAL INTELLIGENCE (AI)

Any technique that enables computers to mimic human behavior. Includes basic "If-Then" logic rules from the 1950s.

THE INNER SHELL MACHINE LEARNING (ML)

Machines that "learn" from data without being explicitly programmed for every rule. Statistical methods.

THE CORE DEEP LEARNING (DL)

Inspired by the human brain (Neural Networks). This is the engine behind ChatGPT, FaceID, and Self-Driving.

THE 3 LEARNING PARADIGMS

1. SUPERVISED

Teacher + Student

Formula: Input + Label = Output.

My Example: Face Verification. I fed the model pairs of photos labeled "Same Person" or "Different Person".

2. UNSUPERVISED

Sorting Lego

Formula: Input Only (Find Patterns).

My Example: TCB News Pipeline. Grouping millions of articles into clusters (e.g., "Onion Price Spike") without knowing the topics beforehand.

Dog Training

09. BUILD LOG

Theory is different from production. Here is a log of the systems I've worked on, the libraries explored, and the technical realities of deploying AI solutions in Bangladesh.

1. FACE RECOGNITION PIPELINE

The Challenge: Implementing robust face recognition that works reliably across different lighting conditions and angles for integration into multiple downstream systems.

The Evolution: It was an iterative process of finding the right backbone:
  • Iteration 1: Dlib (Good baseline, but struggled with scalability).
  • Iteration 2: DeepFace (Better abstraction, but heavy).
  • Iteration 3: InsightFace (ArcFace).

The Verdict: InsightFace (ArcFace) proved to be the most efficient accuracy-wise for our specific use cases and became the standard module for our integrations.
TECH STACK: - Python
- InsightFace (ArcFace)
- REST API Integration
- ONNX Runtime

2. ID DOCUMENT LIVENESS (TCB)

The Context: Used during the scanning of TCB cards to provide subsidized products. The goal was to prevent fraud by classifying input images in real-time.

The Classification: The model had to distinguish between Genuine, Blurry/Unclear, Screen Replay, Photocopy, Critical Spoof, and Incomplete Images.

Model Selection: We trained on a custom dataset and benchmarked several architectures:
  • Swin Transformer
  • DinoV2
  • ViT (Vision Transformer)

The Result: While DinoV2 is powerful, ViT provided the best balance of inference speed and accuracy for this specific deployment environment.
TECH STACK: - PyTorch / Vision Transformers
- Custom Dataset Engineering
- Real-time Inference Optimization

3. BENGALI NEWS INTELLIGENCE

The Objective: To derive market insights by analyzing unstructured data from 40+ daily Bengali newspapers.

The Workflow: 1. Scraping: Daily ingestion of articles from diverse local sources.
2. Clustering: Grouping similar stories to detect trending narratives.
3. Tagging: Automated classification of content.
4. Vector Search: Enabling semantic search to find information from various angles (e.g., "commodity crisis") even without exact keyword matches.

TECH STACK: - Web Scraping (Scrapy/Selenium)
- Clustering Algorithms (K-Means)
- Vector Databases
- NLP (Bengali) - Apache Airflow

4. RAG & CUSTOMER SUPPORT

The Application: Building conversational interfaces for customer support systems that go beyond static scripts.

The Tech: Utilizing RAG (Retrieval Augmented Generation) to ground LLM responses in specific company data. This allows the chatbots to answer queries accurately based on updated knowledge bases rather than hallucinating answers.

TECH STACK: - LLMs (Large Language Models)
- RAG Architecture
- Vector Stores
- Conversational AI

5. CUSTOM VOICE AI (STT/TTS)

The Problem: Standard models (Google/OpenAI) often fail with Bengali dialects (Sylheti, Chittagonian, etc.). True Sovereign AI requires models that understand the local context.

The Work (In Progress): We are actively building Custom STT (Speech to Text) and TTS (Text to Speech) pipelines.

Current Status: Fine-tuning OpenAI's Whisper on a custom dataset of dialect-heavy audio. The main engineering challenge right now is optimizing the Word Error Rate (WER) to handle rapid colloquial speech better than the base models.
TECH STACK: - Whisper Fine-tuning (STT)
- Custom Dialect Datasets
- Low-Rank Adaptation (LoRA)
